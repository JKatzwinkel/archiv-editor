Import und Export von Quellentypen

Für gewöhnlich arbeiten mehrere oder alle Mitarbeiter innerhalb eines Projektes mit dem gleichen Satz an Quellentypen. Damit nicht jeder Benutzer eigens die Quellentypen bearbeiten oder bei Installation des Archiv-Editors auf einem weiteren Rechner neu anlegen muss, gibts es die Möglichkeit sämtliche Quellentypen zu exportieren und auf anderen Rechnern einzuspielen.

Der Export wird über den Quellentypen-Editor (▹Einstellungen▹Quellentypen-Editoröffnen) ausgeführt. Dort findet sich unten links ein entsprechender Button, welcher ein Bestätigungsfenster und darauf ein Verzeichnisfenster aufruft. Nun muss ein Zielverzeichnis festgelegt und der Export kann ausgeführt werden. Dabei wird im angegeben Verzeichnis ein Ordner "‘AEReferenceTemplate-yyyy-mm-dd-hh-mm-ss"’ angelegt (statt der hinteren Buchstabenfolgen erscheint das Datum und die Uhrzeit, z.B. AEReferenceTemplate-2011-08-19-03-08-12), indem sich alle exportierten Quellentypen als einzelne Dateien im xml-Format befinden.

Für den Import eines Quellentypes, muss dieser in der Maske unter ▹Datei▹Importieren als Importtyp angewählt werden. Durch "‘Next"’ wird die Dateiauswahl für den Import aufgerufen. Der zu importierende Quellentyp muss unter "‘Auswählen"’ angegeben und anschließend über "‘Datei laden"’ eingelesen werden. Im nächsten Fenster muss der Import nun unter "‘Finish"’ bestätigt werden. Sollte dies vom Editor verweigert werden, so besteht ein Namenskonflikt, d.h. es existiert bereits ein Quellentyp mit demselben Namen. Dieses Problem kann entweder durch Überschreiben des vorhandenen Types oder Wahl eines Alternativnamens für den zu importierenden Typ gelöst werden.
