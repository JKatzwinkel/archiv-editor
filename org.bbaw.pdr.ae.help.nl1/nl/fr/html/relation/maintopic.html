Beziehungen. Einführung und Konzept

Beziehungen zwischen Personen sind höchst unterschiedlich und ungreifbar zahlreiche. Sie hängen nicht zuletzt von der Perspektive ab, die auf die Personen geworfen wird, deren Beziehungen beschrieben werden sollen. Sind zwei Personen, die an einer Arbeitstelle miteinander arbeiten und sich regelmäßig in der Kneipe treffen Freunde oder doch nur Arbeitskollegen. Wenn ich sie als mehr denn nur Arbeitskollegen ansehe, passt dann das Beziehungsverhältnis Bekannte nicht doch besser. Dies hängt von meinem Verständnis der Beziehung Freund ab. Die Möglichkeiten sind unendlich und sollten daher auch nicht seitens des Repositoriums festgesetzt werden. Auf der anderen Seite ist ein Anspruch der an das Repositorium gestellt wird die Fähigkeit zur Darstellung komplexer und aussagekräftiger Beziehungsnetzwerke. Damit dies Möglich wird muss das System verstehen können, welche Beziehungen einen gewissen Bedeutungszusammenhang besitzen. So macht es in vielen Situationen Sinn die Beziehung vaterVon mit der Beziehung schwesterVon in einem Netzwerk darzustellen, da beides Verwandschaftsbeziehungen sind und die Darstellung eines Familiennetzwerks ein mit Sicherheit häufig anzutreffendes Setting ist. Aus diesem Grund werden Beziehungen im Personendaten-Repositorium in einer 3-teiligen hierarchischen Klassifikation definiert, die es ermöglicht eine solche Zusammenstellung jenseits konkreter Beziehungsnennungen zu vollziehen.

 

Die oberste Ebene der Hierarchie heißt context, darunter kommt die class und dann die Beziehung als relation selbst. Context beschreibt das, was im letzten Absatz als sinnvoller Zusammenhang von einzelnen Beziehungen bezeichnet werden kann. Diese Zusammenhänge konstituieren sich dadurch, dass sie allgemeinhin als ein zusammenhängendes soziales System betrachtet werden, wie z. B. die Familie oder das Berufsfeld. Sinnvoll kann ein solcher Zusammenhang aber auch sein, weil ein bestimmte Fragestellung ihn immer wieder hervorbring wie im Kontext Diskursiv. Hier steht zum Beispiel der Wunsch die Rezeptionsgeschichte des Werks einer Person darzustellen im Fokus. Unterhalb des context kommt die class, die versucht passend zum context die formalen Beziehungstypen dieses Kontextes zu definieren. So gibt es im formal-beruflichen Kontext hierarchische und horizontale Beziehungen. Mein Vorgesetzter befindet sich in einem hierarchischen Verhältnis zu mir, während mein Arbeitskollege als gleichberechtigter in horizontalen Verhältnis zu mir steht. Je nach context kann eine unterschiedliche Typisierung vorliegen und es macht Sinn, erst einmal ein Set an konkreten Beziehungen zu definieren, bevor man eine Typisierung vornimmt. Auf der dritten Hierarchiestufe folgt die konkrete Beziehung selbst. Beziehungen können in den meisten Fällen bidirektional beschrieben werden. Da bei der Definition von Beziehungen nicht sofort klar ist, von welcher Perspektive aus eine Beziehung benutzt wird und diese Perspektive sowieso in der konkreten Erschließung von Personen ständig wechselt, sollte jede Beziehung zweimal, in der je unterschiedlichen Perspektive festgehalten werden. Also z. B. vaterVon und tochterVon.

Das Personendatenrepositorium bringt in seiner Ausgangskonfiguration ein Set an Definitionen von contexts, classes und types mit. Damit wird nicht der Anspruch erhoben, alle existierenden oder für Forscher relevante Beziehungen identifiziert zu haben. Aus diesem Grund kann eine jede Repositoriumsinstanz in seiner Konfigurationsdatei, das mitgelieferte Set verändern oder erweitern. Dennoch sollte dieses Set einen großen Bereich üblicher Beziehungen abdecken. Wenn es nicht dringend notwendig ist empfiehlt sich eine Benutzung der vorgegebenen Beziehungen, da so die Interoperabilität mit anderen Repositoriumsinstanzen gewahrt bleibt. Hierdurch wird es möglich den eigene Datenbestand mit dem anderer zu verknüpfen und damit den Wert der eigenen Daten, nicht zuletzt  für einen selbst, erheblich zu steigern. Aus diesem Grund und um das eben beschriebene Konzept durch Beispiele konkreter zu machen, folgt eine Beschreibung der vom Personendaten-Repositorium vorgegebenen Klassifikationen und Beziehungen.

Der produktive Kontext bezeichnet alle Beziehungen die bei der Entstehung ideeller oder materieller Güter relevant sind, also z. B. ein Roman, ein Musikstück, ein Vertrag, aber auch eine Skulptur, ein Bild.

Realisierende Beziehungen fokussieren auf Beziehungen, die den Produktionsprozess direkt betreffen

 

 

Kollaborativ

 

 

Kollaborative Beziehungen fokussieren auf die Unterstützung der Menschen, die im Produktionsprozess involviert sind, z. B. Ein Zulieferer von etwas

 

 

Formal

 

 

Formale Beziehungen fokussieren auf Beziehungen, die den formalen Rahmen vorgeben in dem ein Produktionsprozess stattfindet, z.B. ein Auftraggeber.
