Editing classifications

Open the classification-editor and select your project, click “Open”. Now you can see your project’s classification system.

The classification system consists of three parts:

a simple list of semantic classification

a hierarchical list of relationship types

Multiple hierarchical lists for markup (for example, for organizations or for person names)

Choose an option from one of the lists. You can navigate through the branched lists by clicking on the plus or minus symbols, or by using double clicks. On the right side, in the text field “Label”, is the name of the category that is also shown in the user interface of the program. Directly below is the text field “Value”, which states the name of the category as it is used by the database. Only administrators can create new classifications and delete or rename existing ones. In addition, there are certain classifications that are obligatory and predetermined by PDR. These cannot be deleted or renamed and are labeled with a key. Every user can adjust which classifications are shown by checking the box labeled “Ignore” (see below for more information).

In the lists that define relationships and markup, it is possible to structure classifications in up to three hierarchical levels. This allows for more exact structuring, for example, the ability to create different categories for professions, such as the larger category “handicraft”, under which the professions “printer” and “shoemaker” can be found.



To create a hierarchically lower classification, the higher classification should be marked in the list on the left hand side of the classification-editor. After it has been marked, enter the name of the new (and hierarchically lower) classification in the text field “New” and then click “Add”.



If a classification has multiple sub-classifications, these can be sorted alphabetically by clicking "‘Sort Children"’ in the right hand field. Alternatively, sub-classifications can be given a number through the button on the right labeled "‘Priority"’ and thus be numerically organized. Lastly, the classifications can simply be rearranged per Drag-and-Drop within the list.

If a classification is no longer needed, it can be deleted by marking it and clicking on “delete”. The name of a classification can be changed by editing the text field “Label”. These rights are for Admins only, non-Admin users can decide if a classification should be hidden or not by checking (or not checking) the box “Ignore”. Hiding classification can allow for a better overview of markup.
