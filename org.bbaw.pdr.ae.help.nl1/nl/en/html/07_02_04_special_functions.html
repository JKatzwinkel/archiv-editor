Special functions

The following functions are generally only for special cases or special needs.

(a) General

Markup of

An aspect often corresponds to more than one person. It must then be clarified which markup belongs to which person. If this isn’t immediately clear, the field “Markup of” provides help.

Example: An aspect reads “Heinrich Müller met Professor Schulze from Göttingen in his hometown Hannover”. While “Hannover” corresponds to Heinrich Müller, “Professor” and “Göttingen” correspond to Professor Schulze.

In this case the correct person can be chosen with “Markup of” and then saved with the general markup.

Keys

With the help of a key, a markup can be provided with an external ID. For example in the above case, the identifier from a place register could be used.

Content

This is a simple list that shows all the texts that exist in the database that have been marked up. This relieves the work of entering unnecessary new markup. Names that are classified as standard names are shown with a green check mark.

(b) Connections and relationships

Connections

Connections bring together aspects and persons.

Normally an aspect corresponds to just one person. For instance, a name usually belongs to one person.

An aspect can also refer to more than one person. If more than one person is described in an aspect, for example, the residence of the couple X and Y, then this connection is retained through this option.

Relationships

In addition, relationships between persons can be explicitly documented. This can be, for example, a familial relationship or a student-teacher relationship.

Select both persons according to the classification of “Subject-Object” and then define the nature of the relationship with the help of the list. The types of relationships are hierarchically structured.

The list of relationship types can be edited with the configuration-editor.

(c) Time and place

Information about the time and place is already defined through the markup of the notification. Here more detailed information can be given that isn’t documented through the markup alone.

The data from the markup is automatically entered here. Normally there is no need to change these entries.

(d) Administrative data

This tab shows general information about the currently selected aspect. Here one can find the internal ID of the aspect and information regarding its creation and recent changes made.
