Basic functions

(a) Allgemein (a) Generale

Semantic classification

The aspect can now be given a classification. This allows the aspects to be categorized within the Archiv-Editor. Every project has its own classification system.

The first aspect of a person should, for practical reasons, be classified as a standard name. Choose from the list the option “NormName_EN”. If the person was created with the wand, then this option has already been chosen.

Normally you would use your own classification system. It is however also possible to use the classification system of another project. You can also select more than one classification.

Content New information about a person can be entered in the lower text field “Content of Aspect - Notification”. This information can originate from a descriptive text or reference, for example “The composer Georg Friedrich Händel was born in Halle in 1685 ”.

The first aspect that is entered should be a name. Names are used to identify the persons and are used to show them in the person list of the Archiv-Editor. Simply enter the preferred name of the newly created person in the text field.

Markup

The content of the text should be marked up–doing this allows the text to be searchable.

Highlight a single word or a phrase of the notification text. Choose one of the options from the Markup-List (found in the middle of the window) which fits to the content of the marked text.

At least “Element” and “Type” should be used. Further markup is optional. To show what has been marked up, click on the button “Set”, found above the notification text. Text that has been marked up will then be highlighted in color.

When creating a standard name, at least the last name of the person should be marked up. This is necessary so that the person can easily be found in the alphabetical list.

(b) References

Every aspect requires a reference. Either a new reference can be created, or one can be chosen from the list of those that already exist.

If you create and save a new reference while editing or creating an aspect, it will be automatically selected for that aspect.

If the aspect has multiple references, it is possible to enter more than one, just click on “More References”.

Internal

The field “Internal” is used to identify exactly where the information used for the aspect can be found within the reference, for example a page or line number. This is provided for cases when this information cannot be documented in the general citation in the reference-editor.

Quality

Every cited reference requires a quality evaluation. Quality refers to the reliability and verifiability of the information. One can choose between “certain”, “probable” and “unsure”. The evaluation only refers to the current aspect being provided by the reference, not to the entire reference.

certain = sure, verified, recognized

probable = largely recognized, not proven

uncertain = unsure, suspected, speculative, not generally recognized, only theoretically possible

Interpretation

Here it is possible to enter an interpretation of an aspect’s content. This can be used when the meaning of the text isn’t clear. The length of the interpretation is limited to 1,000 characters.
