Importing and exporting reference types

Users working on the same project generally use the same reference types. In order to avoid each user having to enter the project reference types on their computer, it is possible to export the types.

To export reference types, open the reference-type editor by clicking (▹Preferences▹OpenReference Genre Editor). In the bottom left corner is a button labled “Export”. After clicking the button, a confirmation window will appear requesting information regarding where the reference types should be exported to. After this has been selected the export can be carried out, and a folder labeled “AEReferenceTemplate-yyyy-mm-dd-hh-mm-ss” will appear in the chosen location in which all the reference types as individual xml files can be found. (the files will be labeled with the date and time, for example: AEReferenceTemplate-2011-08-19-03-08-12).

To import reference types, simply click ▹File▹Import.... Choose “Reference-Template” as the input type and click “Next”. Select the file that is to be imported and click “load file”. After the file loads, the import is confirmed by clicking “Finish”. If the Editor denies the import, this is due to an overlap of reference types. This means that it was attempted to import a reference type with the same name as one that already exists in the editor. If this occurs, the reference type that already exists in the Editor can be overwritten by the new type, or a new name can be selected for the imported reference type.
