Basic window

Once the Archiv-Editor has been started and the user has logged in, the basic program window will open, which looks like the following (allowing of course for differences depending on the data found in the Editor).



There are four different visual areas: (1) Task Menu (2) Icons (3) Tree-view (4) Viewing Area

Task Menu

In the upper area of the program window lies the task menu. Here you can find, as in most programs, functions and options for program settings and tools. All functions that can be reached through the task menu will be designated with a “▹”. For example, ▹File▹Exit. Clicking a button will however be shown with a “▸”

Icons

Also in the upper area of the program window but beneath the task menu, there is a row of colored icons. These icons are for the main functions that are often used while working in the database, such as entering person data or editing already entered data. The function of the icon appears when the mouse passes over it.

Tree-View

At the very left of the program window, beneath the icon list, is the tree-view. This view is made up of a combination of fields. The largest field is the bottommost box, the tree-view itself. Here one finds an alphabetical list of the objects currently in the Editor, such as persons or references. Starting with the first letter of the object, the tree branches out (when enough data has been entered) according to the next letter of the object’s name. Clicking on these letters reveals the objects themselves. Different types of objects can be reached by pressing the appropriate tab.



If an object is marked in the tree-view, its basic data (for persons: first and last name, date of birth, name variations, etc.,) appears in the upper white box (directly beneath the icon list).

Beneath this box is the criteria on which the tree is based. Here, objects can be filtered by changing the criteria, and specific searches can be made. For example, the criteria can be changed so that all persons who have the name “Heinrich” appear, and this list will be found in a new tab in the tree-view. More about searches can be found in chapter 10.



Viewing area

The viewing area takes up the most space, it lies to the right of the tree-view. When an object in the tree-view is selected, all the aspects of this object are showed in the viewing area. These can be categorized and sorted, for example, by semantic classification and the date of the aspect. If the object is a person, then the aspects (such as identity, career, or education) are chronologically sorted. Other criteria, such as reference, place, markup, are also possible.

It is possible to change the perspective of the viewing area. The already described aspect-view is a part of the “main perspective”. In addition there is the identifier perspective, which allows you to search for a person in the internet, and shows information such as their PND, relevant Wikipedia articles or LCCN. The concurrence perspective finds and eliminates double entries. More information on this subject can be found in chapter 9.
